# 图片优化通用指南

这份文档用于记录通用的图片和媒体加载优化方法，不绑定具体项目、页面或组件。

## 1. 首屏关键图片优先加载

首屏可见的核心图片应该优先加载，例如人物主视觉、产品主图、品牌 Hero 图。

推荐写法：

```tsx
<img
  src="/path/to/hero.png"
  alt="Hero image"
  loading="eager"
  decoding="async"
  fetchPriority="high"
/>
```

适合使用在：

- 首屏 Hero 图。
- 首屏人物图。
- 首屏产品图。
- 首屏品牌视觉图。

作用：

- 让浏览器更早请求关键图片。
- 减少首屏图片延迟出现。
- 提升用户对首屏完成度的感知。

注意：

- 不要给所有图片都加 `fetchPriority="high"`。
- 一个页面通常只给 1 到 2 张真正关键的首屏图片设置高优先级。

## 2. 非首屏图片懒加载

首屏之外的图片应该使用懒加载。

推荐写法：

```tsx
<img
  src="/path/to/image.png"
  alt="Image description"
  loading="lazy"
  decoding="async"
/>
```

适合使用在：

- 页面下方内容图。
- 卡片列表图片。
- 奖项、案例、活动、媒体报道等列表图片。
- 轮播图中非首屏直接可见的图片。
- 详情页正文里的插图。

作用：

- 避免打开页面时一次性请求所有图片。
- 减少首屏带宽压力。
- 减少浏览器同时解码大量图片造成的卡顿。

## 3. 控制轮播里的图片数量

轮播、滑动卡片、图片墙等区域不要一次性把所有图片都放进主舞台。

推荐策略：

- 只渲染当前图、上一张、下一张。
- 远离当前视图的图片可以不渲染，或只保留轻量占位。
- 当前图可以使用较高优先级。
- 非当前图使用普通优先级或懒加载。

示意：

```tsx
const visibleItems = [previousItem, currentItem, nextItem];
```

作用：

- 减少 DOM 节点数量。
- 减少图片解码数量。
- 降低轮播区域内存占用。
- 保持动画更稳定。

## 4. 长滚动图片动画要限制计算

滚动叙事、瀑布屏、sticky 图片堆叠等效果，通常会根据滚动位置更新图片状态。

推荐策略：

- 使用 `requestAnimationFrame` 合并滚动更新。
- 使用 `IntersectionObserver` 判断区域是否接近视口。
- 区域离开视口后暂停滚动计算。
- 只更新必要状态，例如当前索引、进度值、遮罩比例。

示意：

```ts
let frameId: number | null = null;
let isNearViewport = true;

function scheduleUpdate() {
  if (!isNearViewport || frameId !== null) return;

  frameId = requestAnimationFrame(() => {
    frameId = null;
    updateScrollProgress();
  });
}
```

作用：

- 避免每一次滚动事件都触发布局计算。
- 减少页面远离动画区域时的无效工作。
- 提高长页面滚动流畅度。

## 5. 图片揭示动画优先使用 CSS

图片切换、揭示、遮罩类动画，优先使用 CSS 属性完成。

可选方案：

- `opacity`
- `transform`
- `clip-path`
- `mask`
- `filter`

示例：

```tsx
<div
  style={{
    clipPath: `inset(0 0 ${clipBottom}% 0 round 24px)`,
    transform: `translateY(${translateY}px)`,
  }}
>
  <img src="/path/to/image.jpg" alt="" loading="lazy" decoding="async" />
</div>
```

注意：

- 动画过程中尽量避免频繁修改宽高。
- 优先修改 `transform`、`opacity` 这类更适合动画的属性。
- 使用 `will-change` 要谨慎，只给真正持续动画的元素添加。

## 6. 固定图片容器尺寸

图片加载前后不要让页面布局跳动。

推荐做法：

- 使用固定比例容器。
- 使用 `aspect-ratio`。
- 对卡片图、封面图、海报图设置稳定宽高区域。
- 图片本身使用 `object-fit` 控制显示方式。

示例：

```tsx
<div className="aspect-[16/10] overflow-hidden">
  <img
    src="/path/to/card.jpg"
    alt=""
    loading="lazy"
    decoding="async"
    className="h-full w-full object-cover"
  />
</div>
```

常见选择：

- `object-cover`：适合照片、背景图、视觉卡片。
- `object-contain`：适合海报、截图、证书、Logo，避免内容被裁切。

## 7. 同一行卡片图片高度要统一

如果一行有多张卡片，并且图片比例不同，需要避免每张卡片高度各自变化。

推荐策略：

- 卡片外层使用等高网格。
- 图片区域使用统一高度或统一比例。
- 如果必须保持图片原比例，可以按当前行最高图片作为统一图片区域高度。
- 较短图片在统一高度内垂直居中。

作用：

- 保持卡片列表整齐。
- 避免同一行图片高度参差不齐。
- 适合奖项、案例、作品集、媒体报道等卡片墙。

## 8. 固定素材和上传素材分层

图片资源建议分成两类管理。

### 固定素材

适合：

- Logo。
- 图标。
- 默认背景图。
- 默认 Hero 图。
- 长期不变的设计素材。

建议位置：

- 项目源码资源目录。
- 公共静态资源目录。

特点：

- 跟随代码版本发布。
- 适合构建期处理。
- 不适合频繁由运营人员替换。

### 上传素材

适合：

- 后台上传图片。
- 活动图。
- 新闻图。
- 奖项图。
- 用户内容图片。
- 上传视频。

建议位置：

- 运行期上传目录。
- 对象存储。
- CDN 源站。

特点：

- 不应该放进源码资源目录。
- 不应该每次换图都重新构建项目。
- 更适合后续接对象存储和 CDN。

## 9. 视频只预加载元数据

大视频不要在页面打开时直接完整预加载。

推荐写法：

```tsx
<video
  src="/path/to/video.mp4"
  controls
  preload="metadata"
/>
```

作用：

- 只预加载视频尺寸、时长等基础信息。
- 避免大视频抢占首屏图片和接口带宽。
- 降低移动端流量消耗。

## 10. 部署时不要带无效缓存

构建缓存通常只用于本地或 CI 加速，不应该进入线上运行包。

建议排除：

- 构建缓存目录。
- 本地开发缓存。
- 临时截图。
- 调试浏览器 profile。
- 无用源文件备份。

作用：

- 减少部署包体积。
- 减少上传时间。
- 降低服务器磁盘占用。

## 11. 进一步优化方向

如果图片仍然加载慢，可以继续做下面这些优化。

### 自动压缩

上传时自动压缩超大图片。

建议：

- 限制最大宽度，例如 1920px 或 2560px。
- 控制 JPEG/WebP 质量。
- 保留原图作为备份，前端默认使用压缩图。

### 自动生成缩略图

卡片、列表、后台文件管理中不应该直接使用原图。

建议生成：

- `small`：列表缩略图。
- `medium`：卡片图。
- `large`：详情页图。
- `original`：原图备份。

### WebP / AVIF

现代浏览器优先使用 WebP 或 AVIF。

建议：

- 原图保留。
- 同时生成 WebP。
- 对高访问量图片再生成 AVIF。

### OSS + CDN

图片访问量上来后，建议把上传资源放到对象存储，并通过 CDN 分发。

基本结构：

```text
用户上传 -> 服务器接收 -> 对象存储 -> CDN 域名 -> 前端访问
```

好处：

- 减少服务器带宽压力。
- 多地区访问更快。
- 图片和应用服务解耦。

### 静态缓存

对不会频繁变化的图片设置较长缓存。

建议：

- 带 hash 的构建资源可以长缓存。
- 上传文件如果文件名带唯一 ID，也可以长缓存。
- 会覆盖同名文件的资源不要设置过长缓存。

## 12. 推荐落地顺序

1. 首屏关键图片设置 `eager + fetchPriority="high"`。
2. 非首屏图片统一设置 `lazy + decoding="async"`。
3. 所有图片容器设置稳定比例或高度。
4. 轮播只渲染当前附近图片。
5. 长滚动动画使用 `requestAnimationFrame` 和 `IntersectionObserver`。
6. 后台上传区生成缩略图。
7. 上传图片自动压缩。
8. 生成 WebP / AVIF。
9. 上传资源迁移到 OSS。
10. OSS 前面接 CDN。
11. 给静态资源配置合理缓存策略。
